Link your github page to your domain

For the developers it is quite common now a days to create there blogs,websites using the same tool which they use to version control their code. Yes i am talking about git.You can create your static github pages using github pages and then using the Static site generators like hugo or jekyl you can build your website.Being on the github these websites are accessible using your [githubranchname].github.io But usually you would like people to access your website using your preferred domain name. right?

For that very purpose you can follow these simple steps. First thing you need to do is make your git repository aware of your domain name.

this can be done by adding CNAME file in your repository at the root level. In this file add your domain name. i.e. if your domain name is www.mydomainname.com. just add one line

www.mydomainname.com

in your CNAME file. make sure to right CNAME in uppercase.

Now it’s time to change Settings in your domain provider

I will take example of go daddy

In your go daddy account select My Products option from the drop down menu at the top right corner

You will see list of the products. One of them will be the Domains.

Click Manage button against the domains.

you will see all of your domains listed. Just go to the DOMAIN DETAILS page by selecting Manage DNS from your preferred domain.

Here you have to modify two things

A record and the CName alias

For editing A record enter ip address of your github page.

you can find ip address of your github page using following command on your terminal

 dig yourgitgubrepo.gitgub.io

ip associated with A record is the ip of your github page. Add this ip to your A record in the Points To section of @ host .

For adding alias CName. add one more entry for your host add www in host field and your githubrepo.github.io in the Points to field. And then save the setting these changes will take at most 24 hrs to get reflected.If you are lucky you will see them in couple of minutes.

you can cross verify these changes by applying following Command

 www.yourdomainname

If you are able to see the ip,you added with the A record you are good to go. otherwise you would have to wait for little longer.

once these changes are reflected. you can just use your domain name to access your website and share with your viewers.
